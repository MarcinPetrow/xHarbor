# xDashboard

`xDashboard` is the reporting surface for the platform. It reads cross-module data from `xGroup` and `xBacklog` and turns it into operational summaries.

![xDashboard overview](../../assets/screenshots/xdashboard-overview.png)

## Responsibilities

- executive overview
- team load and delivery risk views
- recent workflow activity
- completed work snapshots

## Main views

- `Overview` for high-level platform state
- `Risks` for blocked work and delivery warnings
- `Team Load` for capacity and workload distribution
- `Activity` for comments and recent task changes

## Notes

`xDashboard` is intentionally read-oriented. It is the place to inspect delivery health built from shared platform data rather than a place to change workflow state directly.
