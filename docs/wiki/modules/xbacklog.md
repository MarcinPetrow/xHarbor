# xBacklog

`xBacklog` is the planning and execution surface for delivery work. It consumes workspace structure from `xGroup` and keeps projects, tasks, comments, and change history in one focused flow.

![xBacklog board](../../assets/screenshots/xbacklog-board.png)

## Responsibilities

- project ownership by team
- task board with `new`, `in_progress`, and `done`
- task detail with description, assignee, and timestamps
- comments and task history
- board filtering and drag-and-drop workflow

## Main views

- `Board` for day-to-day task movement and dedicated task entry points
- `Projects` for project list, create, edit, and delete flows
- `Comments` for recent workflow discussion

## Notes

`xBacklog` is designed as a compact execution tool rather than a general dashboard. The board stays focused on task movement and task selection, while project and task create/edit flows are handled in dedicated screens instead of being mixed into list pages.
